There is one protocol, many clients

...stakeholders like miners, developers, users

...and only one main network

How do the stakeholders in this decentralized network coordinate an upgrade?

Key constraints for network upgrades

  • Software and network complexity
  • Coordinating client updates
  • Implicit social contract
  • Opposition to rolling back or altering state
  • Reputation must be upheld

Principle: Cautiousness

  • Extensive in-house testing
  • Deploy to multiple testnets
  • Bounties for finding bugs, vulnerabilities
  • Client diversity

Principle: Coordination

  • All Core Devs calls
  • Discussions about proposals: EIPs
  • PM -> "Ethereum Cat Herders"

Principle: Cadence

  • Major releases are rare
  • Each feature specified in an EIP
  • Each release contains a set of EIPs

Workflow of a successful Core EIP

Initial Submission

  • Fork the repo in GitHub.
  • Add your EIP to your fork. Use the EIP template.
  • Submit a Pull Request & Status Change to the repo.

What can fork the protocol?

Changes to the core components

  • Ethash: Proof of Work
  • Txn, block format
  • Fee schedule
  • RLP: object serialization
  • Trie: modified Merkle Patricia tree
  • Method of signing transactions
  • EVM: virtual machine
  • Wire Protocol

Release Case: DDoS Response

Fall 2016

  • Miners mainly on geth, cpp
  • Network was becoming more diverse, some parity adoption

DDoS Attacks

  • Several EVM operations were underpriced by 100x
  • Attacker flooded w/ txns targeting opcodes
  • Non-parity nodes unable to create blocks

Parity "Kept the Lights On"

  • Developed in rust lang
  • Profiled and optimized
  • Was more able to absorb the attack
  • Miners, key operators migrated over

DDoS Response Releases

v5 - Tangerine Whistle
EIP-608: Hardfork Meta
EIP-150: Gas cost changes for IO-heavy operations

v6 - Spurious Dragon
EIP-607: Hardfork Meta
EIP-155: Simple replay attack protection
EIP-160: EXP cost increase
EIP-161: State trie clearing (invariant-preserving alternative)
EIP-170: Contract code size limit

Peek into v5's EIP-150

Simple replay attack protection

Specification
If block.number >= FORK_BLKNUM, then:

Increase the gas cost of EXTCODESIZE to 700 (from 20).
Increase the base gas cost of EXTCODECOPY to 700 (from 20).
Increase the gas cost of BALANCE to 400 (from 20).
Increase the gas cost of SLOAD to 200 (from 50).
Increase the gas cost of CALL, DELEGATECALL, CALLCODE to 700 (from 40).
Increase the gas cost of SELFDESTRUCT to 5000 (from 0).
If SELFDESTRUCT hits a newly created account, 
  it triggers an additional gas cost of 25000 (similar to CALLs).

https://eips.ethereum.org/EIPS/eip-150
